MyApps

Background

MyApps Admin Platform is designed for developers at Walmart Supply Chain org to speed up product launch process. This is a cloud based platform to manage all the code package variations grouped by distribution center sites. This will allow developers have visuability into history edits, reuse code and never lose track of edit details. This platform is being piloted and will become the go-to place for 40+ dev teams at Walmart Supply Chain Org to config code packages and destination sites.




My role

This is a one-year project, where I lead the end-to-end design process. I colloborated closely with a product manager, 10+ developers and a design manager on a weekly basis. I planned and conducted interaction designs according to tech sprint plans. I facilitated 5+ user review sessions, 10+ design review sessions and 3 main tech review sessions to gather stakeholders' feedback. I brought different concepts to the table in weekly UX meetings to address competing priorities. I documented all the use cases through InVision prototype along with a UI library and an Endnote design specification.




Outcome

There are two parts of the project, MyApps Web for warehouse associates and MyApps Admin Portal for developers. The reskinned web version is released to 200+ distribution centers with 50+ apps implemented. The admin portal is being piloted in 2021 Q3. I shared the design process with the team and my manager commented:"Seriously great job on MyApps - you've pushed it a long way and made a complex process very simple".

Learnings

Move fast and do things right

The business goal of this project was to launch the platform and then gather feedback through this MVP. I needed to create key flows in a timely manner so that devs can start building the MVP. After I created the first version, I brought it to the design review and showed the concept with 10+ users. After reading through the comments received, I discovered one common user problem and it was directly related to one of the five main tasks, which will heavly influenced our user experience when the MVP is launched.

To find potential solutions, I created some proposed iterations, brought it to the design lead and discussed if we could improve the flow before development cycle. After winning design managers' approval, I scheduled meeting with tech lead, checked their sprint deadline and identified a potential time frame to finish this iteration. The new concept was validated with end users again, and there were no users complaining about the problem any more. What I learned is that when there were potential problems, advocating for right solutions in a timely manner can push the project forward.




Increase consistency with design systems, but push it forward

This project was divided into two phases, splitted by the launch of Living Design System _ Associate Version. I focused on building key flows with existing UI library, then applied new living design system to this platform. Having a design system makes our work easier with established pattern. But it doesn't mean stop innovating. One of the site selector I designed was selected by the team and become part of the new design system.




Communicate timeline, deliveries and details

After onboarding to the team, I created a journey map to communicate the key tasks and user flows. When the tech sprint timeline was created, I also created a UX timeline to specify the delivery date. For every handoff cycle, I send out the prototype links, detailed specifications and UI library. These bring the product, design and development team on the same page about project progress.




Reflection

This has been a great experience working directly with developers from brainstorming to development. The design team and tech team have been incredibly supportive. The whole team was always flexible and open-minded. Over-communication is never a problem in this team, more like a habit to pursue something better.

One thing I think can be improved in this project is the strategy to prioritize user feedback. I decided to optimize some flows based on my experience or personal insights. If I have the opportunity to redo the project, I would use more quantitive methods to communicate the importance.
