MyApps

One stop shop for product launch at Walmart Supply Chain (NDA)

Background

MyApps Admin Platform is designed for developers at Walmart Supply Chain org to launch, customize and manage code files. The user pain point we solved in this project is that releasing apps and webs to Walmart Distribution Centers can take a long period to customize code files on a site-by-site basis, as every Distribution Center differs in business, hardware, staff and policy. When fully implemented, it will create a more efficient and transparent work environment for Supply Chain tech teams.




My role

I lead the product design from brainstorming to fuly adopt Living Design, the Walmart Design System. We went through 2 main versions before implementation. Designers, product and developers meet on a weekly basis. In order to ship the product in a timely manner, I designed the product first. When Living Design was officially launched, I took a month to apply the standard design pattern to it.

Intake

Evaluate product through users' eye

What I learned as a project is that we shouldn't limit ourself to just create what PMs/teams told us to. When designng this project, we always found inspiring ideas from our end users. There are flows that can be simplified, redesigned or completely get rid of. For example, we removed a lock logic as we discover it is not consistent with users past experience. Through user testing, we also discover that a side by side comparison view is a great way to help users confirm and think.




Design systems are guidelines, not answers

One of the biggest challenge is about design system. When the project starts, the Living Design System_ Associate Version was being built. However, this product needed to be designed as soon as possible. So I started building the platform with the approved component first. There is a page where request users to do multi-select from over 200 sites. I used a dropdown according to the Living Design System, but then user testing revealed that users hate this long scroll to find what they need to select or what they have already selected.

I decided to customize this element. I meet with the design team and discussed why it is important to customize this site selector in MyApps. The design was approved and the site-selector concept is also selected as one of the new component in the next Living Design System to help other teams who had received similar user feedback.




Improve development experience when handoff

As we know that there would be a reskin process for the development team after the new Design System was released, in order to reduce the workload for the development team, I decided to create a document for all the components we used, listing out all the instances and scenarios. I communicated early with the dev team that if we could build the platform by creating reusable component, it would be easier when we started to work on the visual updates. This strategy works out well. The front end developer used these guides a lot and we met no obstacles when we expressed the need to reskin it.

When the new design system is released, I carefully compared what we already have with the design system. Then I updated the design docuemnts for every elements we have used. As a result, developers do not need to find and compare elements from different pages. It provide a place for developers to find all the instances for different scenarios.




And...

This has been a great experience working directly with developers from brainstorming sessions. The design team and tech team have been incredibly supportive as we go through the whole process. The whole team was always flexible and welcome to talk, even if it was just a small idea. Over-communication is never a PROBLEM, more like a HABIT to always pursue something better. I would't be able to push the design far without their help.
