Voice Emojis

The Challenge

Low Efficency in Audio Message Processing.

When users use current audio message input, assistive emotion expression tools like emoji are hard to use. Users need to stop audio recording process and switch to a traditional keyboard input method. They need to stop recording message, tap on the emoji selection panel where everything is small, recognize the emoji icons and select the right one.

This task is extremely difficult for the ones who don’t have both of their hand free or on a moving action. We aim to design an audio interface to help them use the assistive emotion expression tool like emoji on the audio messaging apps.

01 User Research

Emotion Buffer or Polisher Lacked in Audio Messages.

We compared several user chating history and discovered that when users are using audio messages, usage of emotion buffer or polisher like emoji are way much less than traditional text message.

User Personas

Based on interview results, I draw a userstory board to share with four software engineers to pitch the identified user behaviors. To our suprise, most of the users don't search for parking ahead of time. Rather, they find parking while they were behind the wheel.

Design Criteria

We aim to use the auditory user interface that involves speech-based input by the user to generate non-speech output (emoji) from the system. The design requirements we generated from persona and previous research include

1) The design will be a seamless user experience;
2) The design will be intuitive enough and easy to learn;
3) The design will be interesting enough and add joy to the texting process.

02 Design Solution

Wake Up Word

With these scenarios in mind, we brainstormed several wake-up mechanisms as follows. Through discussion, we discovered that Concept 1 high accuracy demands in recognizing the Wake-up word. When an recogniztion failure happens, the wake up word will be recorded as part of the audio message which will make error fixation hard for users. Concept 3 is more intuitive than concept 2 since it minimizes user finger movements and won't interrupt natural user audio message input process. Users can switch between audio message record process and emoji recognize process more easily.

Utterance Word

Aside from adding an emoji to an audio message, users also need to delete an emoji, replace an emoji or the system can predict an emoji according to iOS emoji usage guidelines. We identified the following utterances to suit for these typical usage scenarios.

Emoji Activation Word

To cater for the needs of the largest population to input various audio messages, we considered three alternative voice input methods to ensure all the users can activate the right emoji with their language habits. * Machine learning technology can be used to meet this goal.

1) Names of emoji, eg. "Happy Face."
2) Emotions related to the emoji, eg. "I am super happy now."
3) Visual Features of the emoji, eg. "Emoji with open mouth."
4) Sound impressions of the emoji. eg. "Hahaha."

03 Interface and Prototype Design

Main Flow

In the process of wireframe, we discovered that the system needs to put users in control when an emoji is recognized. An emoji wait status above the record button is added, providing users ability to judge and proceed with either confirm, change or repeat. When the emoji is added, it will jump to the message box waiting to be sent out.
