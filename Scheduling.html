Scheduling

Background

Transportation managers manage Walmart driving staff, making sure trailers leave distribution centers and arrive at stores on time. This project aims at optimizing one of the admin tasks so that managers can focus more on daily shipping load. Product teams discovered similiar user pain points in every local office, and I helped the team to build a scalable and user-friendly solution.




My role

I colloborated with a product manager and my design manager for 5 months from research, iteration to validation. I created concepts and wireframes to discuss solutions. I iterated three times to design a standard but flexible flow for two types of operation methods. In order to get user buy-in, I designed with real data on tablet and desktop for both managers and drivers.




Outcome

Through five months work, I designed two prototypes for managers and drivers for the proposed user journey. The concept helped to won approval from leadership role and is planning to launch. Learnings

Leading users through a standard but flexible process

After three iterations, I finaly find a way to use a linear process to replace all the excel files. The process is chunked in a way that users can feel they are making progress. A wizard UI clarifies where users are and what are next. The title and subtext help users to confirm expected actions. Some steps are optional and can work in two different operation methods. In this way, local office will be able to improve bidding process and get rid of excel files without changing operation habit.




Design with real data

I used the real schedule data from an office when designing mid fidelity prototype. This revealed edge cases undercovered by dummy data: Not all offices have every type of programs, and the previous dashbaord design failed to display unavailable programs. I spend a week redesigning the dashboard. With the new design, managers will be able to understand if an empty slot means not available or fully booked.




Design for all touchpoints

Drivers are having limited access to live bidding progress right now. In order to get managers buy in, we created a tablet UI for drivers using dark theme UI. We presented both tablet UI and desktop UI to managers to show how the system works. This cleared many questions and help to gain user buy-in.




Documentation and transfer work

We transferred this work to another design team due to org change. I created a process book to document all the design decisions we made. This explained the marketing standing, the difference in operations, what we have decided to do and not to do. I send out the process book to the designer which helped them onboarding with this project quickly.
